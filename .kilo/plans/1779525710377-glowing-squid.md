# Training Pipeline for Kaggle - Foresight Model

## Overview
Train the Foresight LLM on Kaggle's free GPU infrastructure using the collected training data (7,093 training samples, 35 validation samples).

## Prerequisites
- Kaggle account (free tier provides 30+ hours of GPU per week)
- Kaggle API token configured locally

## Step-by-Step Guide

### 1. Prepare Kaggle Notebook

Create a new notebook on Kaggle with these settings:
- **Accelerator**: GPU (P100 or T4)
- **Environment**: Python 3, CUDA enabled
- **Storage**: ~15GB for model weights

### 2. Upload Training Data to Kaggle

Option A: Copy from local to Kaggle dataset
```bash
# Install Kaggle CLI
pip install kaggle
kaggle datasets init
# Configure API token at ~/.kaggle/kaggle.json

# Create dataset
kaggle datasets create -p ./data
```

Option B: Include in notebook as attached files:
- `data/raw_events.jsonl` (2,500 events)
- `data/train_dataset.jsonl` (7,093 samples)
- `data/val_dataset.jsonl` (35 samples)

### 3. Kaggle Notebook Cells

**Cell 1: Install Dependencies**
```python
!pip install -q transformers peft trl datasets bitsandbytes wandb

import os
os.environ["WANDB_DISABLED"] = "true"  # Disable if no wandb account
```

**Cell 2: Load and Verify Data**
```python
from datasets import load_dataset

train_ds = load_dataset("json", data_files="data/train_dataset.jsonl", split="train")
val_ds = load_dataset("json", data_files="data/val_dataset.jsonl", split="validation")

print(f"Train: {len(train_ds)} samples")
print(f"Val: {len(val_ds)} samples")
```

**Cell 3: Configure Training**
```python
from transformers import TrainingArguments
from foresight.training.config import ForesightTrainingConfig

config = ForesightTrainingConfig(
    model_id="unsloth/llama-3-8b-Instruct-bnb-4bit",  # 4-bit quantized version
    output_dir="/kaggle/working/foresight-model",
    per_device_train_batch_size=2,  # Kaggle GPU memory limit
    gradient_accumulation_steps=16,
    num_train_epochs=3,
    learning_rate=2e-4,
    max_seq_length=2048,  # Reduce for memory
)
```

Note: Consider using smaller model for Kaggle:
- `unsloth/llama-3-8b-Instruct-bnb-4bit`
- `mistralai/Mistral-7B-v0.1` with bitsandbytes
- `Qwen/Qwen1.5-7B`

**Cell 4: Train Model**
```python
from foresight.training.train import train_foresight

train_foresight(config, "data/train_dataset.jsonl", "data/val_dataset.jsonl")
```

### 4. Memory Optimization for Kaggle

Kaggle T4/P100 has ~16GB VRAM. Use these optimizations:

```python
config = ForesightTrainingConfig(
    per_device_train_batch_size=1,
    gradient_accumulation_steps=32,
    max_seq_length=1024,  # Truncate context
    lora_r=8,  # Smaller LoRA rank
)
```

### 5. Alternative: Use Unsloth for Faster Training

```python
!pip install -q unsloth

from unsloth import FastLanguageModel

model, tokenizer = FastLanguageModel.from_pretrained(
    model_name="unsloth/llama-3-8b-bnb-4bit",
    max_seq_length=2048,
    load_in_4bit=True,
)

model = FastLanguageModel.get_peft_model(
    model,
    r=8,
    lora_alpha=32,
    target_modules=["q_proj", "k_proj", "v_proj", "o_proj"],
    use_gradient_checkpointing=True,
)
```

## Discoveries

**Architecture Patterns:**
- Existing codebase uses `foresight.training.train.train_foresight()` which expects `ForesightTrainingConfig` dataclass with specific fields
- The temporal sequencing uses `TemporalSequencingEngine` that creates sequences by iterating through all events - this creates 4,960 raw sequences from 2,500 events due to multiple horizon predictions per cutoff
- Dataset augmentation multiplies sequences by ~5x (3,962 → 19,805) via jitter and dropout variants

**Gotchas:**
- Kaggle GPUs (T4/P100) have ~16GB VRAM - need smaller batch sizes and gradient accumulation
- The default Llama-3-8B config uses 8192 max_seq_length which needs reduction for Kaggle memory
- Validation set only has 35 samples because sequences need future events within horizon - most cutoff points don't have future events in synthetic data spanning 2022-2026

**Existing Code References:**
- `foresight/training/train.py` has `format_foresight_sample()` using chat template with system message "You are Foresight, a predictive model forecasting AI industry events"
- LoRA target modules include both attention (`q_proj`, `k_proj`, `v_proj`, `o_proj`) and MLP (`gate_proj`, `up_proj`, `down_proj`) layers
- Model uses 4-bit NF4 quantization with double quant and fp16 compute

## Implementation Notes

**Conventions:**
- Use `sys.path.append()` for local imports in Kaggle environment
- Set `WANDB_DISABLED=true` if no Weights & Biases account
- Output models to `/kaggle/working/` for persistence

**Dependencies Between Steps:**
- Must run collect script first before build to have enough events for sequences
- Build script requires >= 20 past events per sequence (filter applies)
- Training requires datasets in exact JSONL format expected by `load_dataset("json")`

**Pitfalls to Watch:**
- The existing `raw_events.jsonl` has some events with `_ts` field added by build script - ensure fresh events don't have this
- Synthetic events use random timestamps spread across 2022-2026, reducing valid validation sequences
- Original events use "Z" suffix while synthetic uses "+00:00" - both parse correctly via `parse_timestamp()`
