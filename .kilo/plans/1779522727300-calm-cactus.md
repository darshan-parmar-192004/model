## Implementation Plan

### Phase 1: Fix Critical Issues

#### 1.1 Fix SEC EDGAR URL Construction (`foresight/ingestion/macro_signals_scraper.py`)
**Problem:** Filing document URLs return 404. The current code constructs URLs that don't match SEC's actual document structure.

**Solution:**
- SEC filing URLs follow pattern: `https://www.sec.gov/Archives/edgar/data/{cik}/{accession}/{accession}-index.html`
- Actual document URLs need to be extracted from the filing's index page
- Fix the `_analyze_filing_for_keywords` method to properly extract document links

**Changes needed:**
1. Remove the `-index.htm.txt` suffix handling (lines 203-209)
2. Properly parse the filing index page to find actual document URLs
3. Handle the accession number format (remove dashes)

#### 1.2 Fix Kaggle T4 GPU Compatibility (`foresight/training/config.py` and `train.py`)
**Problem:** Uses `bf16` precision and `flash_attention_2` which aren't supported on T4.

**Solution:**
- In `config.py`: Change `bf16: True` to `fp16: True` in `to_training_args_dict()`
- In `train.py`: Change `torch.bfloat16` to `torch.float16` in:
  - `BitsAndBytesConfig` compute_dtype
  - Model loading `torch_dtype`
  - Remove `attn_implementation="flash_attention_2"` (T4 doesn't support it)
- Add fallback logic for attention implementation

#### 1.3 Fix Event ID Consistency
**Problem:** `known_events.json` uses `event_id` field but pipeline code checks multiple fields inconsistently.

**Solution:** Ensure consistent field name across all scrapers.
